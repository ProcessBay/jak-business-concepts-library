# Create-Deliver-Capture Value

## Definition

Pereira's three-component definition of a business model: every business must create value (produce something worth more to customers than it costs to deliver), deliver value (get the value into customers' hands via channels and relationships), and capture value (extract a portion as revenue). All three are necessary; missing any one breaks the model. Create without deliver = no customer access; deliver without create = giving away nothing; create + deliver without capture = beautiful failure (the firm doesn't survive). Pereira frames the entire 20th-vs-21st-century analysis around showing how each of 13 industries shifted these three components when digital disruption arrived.

## Key attributes

- **Three required components** — Create, Deliver, Capture
- **Sequence matters** — value must be created and delivered before any can be captured
- **Customer-side acceptance** — "delivery" is really "customer acceptance"; the customer decides whether your value is real
- **Capture % is strategic** — too little = unviable business; too much = customers feel exploited
- **Each industry's evolution** — usually one or more components shift dramatically when disruption arrives
- **Customer-share-of-value** — modern firms often share more value with customers than 20th-century firms did (Dropbox example)

## When to use

- Diagnosing a struggling business — which of the three is failing?
- Designing a new venture — articulate all three components explicitly
- Investor pitch — clear C-D-C story is core to understanding the business
- Strategic planning — examine which of the three is being disrupted
- Industry analysis — map an industry's 20th-vs-21st-century shifts

## When NOT to use

- For pure operational decisions disconnected from business model
- As a substitute for richer frameworks (BMC) — C-D-C is a lens, not a complete tool

## Common pitfalls

- Optimizing one component while neglecting another
- Focusing on capture (revenue) without understanding create (real value)
- Confusing delivery with marketing (delivery is the actual transfer of value)
- Assuming customer accepts value because firm declares it (acceptance is customer's choice)
- Capturing too much value, alienating customers (high take-rate creep)

## Key metrics

- Value created (customer-perceived dollar value of offering)
- Conversion rate (delivery effectiveness)
- Value captured as % of value created (firm's slice vs. customer's slice)
- Gross margin (a proxy for value capture efficiency)

## Decision questions

- For our business: precisely how do we create, deliver, and capture value?
- Where is our current model weakest of the three?
- Could we share more captured value with customers without breaking the business?
- How is each component being disrupted in our industry?

## Canonical examples

- **Dropbox** — Create: solve multi-device file sync; Deliver: free + paid online subscription; Capture: freemium with 2GB free + paid tiers; pages 9-10
- **Press 20th C** — Create: journalists write; Deliver: paperboys; Capture: ads in print (page 12)
- **Press 21st C** — Create: users on Google/Facebook; Deliver: platform; Capture: digital ads (page 13)
- **Music 20th C** — Create: physical store catalog; Deliver: customer chooses; Capture: album sales (page 20)
- **Music 21st C (Spotify)** — Create: streaming catalog; Deliver: web/mobile app; Capture: all-you-can-eat subscription (page 21)
- **Travel 20th C** — Create: physical travel agency; Deliver: travel agent; Capture: ticket/package fees (page 22)
- **Travel 21st C (Expedia)** — Create: aggregator/comparison tool; Deliver: self-service web; Capture: credit card (page 22)
