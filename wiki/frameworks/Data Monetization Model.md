# Data Monetization Model

## Definition

The Data Monetization Model captures value by treating data itself as a sellable asset — through direct data sales, data-licensing subscriptions, insights/analytics services derived from data, or AI models trained on proprietary data. Pereira positions it as one of the major AI business models: as AI capability commoditizes, the data feeding AI becomes the differentiator. Two structural patterns: (1) Direct Monetization — sell or license data to third parties (often anonymized/aggregated for privacy); (2) Indirect Monetization — use data to build AI products/services that customers buy. Major players include credit bureaus (Experian, Equifax), financial data providers (Bloomberg, S&P), telco data brokers, retail loyalty programs, and increasingly platforms that aggregate user behavior data.

## Key attributes

- **Data as asset** — treated like inventory; value depends on quality, freshness, exclusivity
- **Direct vs Indirect monetization** — sell data OR use data to build sellable products
- **Privacy/regulatory constraints** — GDPR, CCPA, sector-specific regulations heavily limit options
- **Network effects in data** — more data → better products → more users → more data
- **Data quality > volume** — clean, structured, recent data beats massive dirty datasets
- **Long-tail use cases** — same dataset can serve many buyers with different needs
- **Exclusivity premium** — proprietary data commands higher prices than aggregated public data

## When to use

- When firm sits on proprietary data with external market demand
- Industries where data is genuinely scarce or expensive to collect
- B2B where customers need insights from data they can't collect themselves
- When the firm can ensure privacy/regulatory compliance
- AI applications where training data is the moat

## When NOT to use

- When data is commodity-like (publicly available alternatives exist)
- In categories with severe privacy/regulatory constraints making monetization risky
- When data collection methods would damage customer trust if disclosed
- When the firm lacks data governance to safely productize data

## Common pitfalls

- Privacy violations (Cambridge Analytica-style scandals destroy trust + invite regulation)
- Selling data that undermines the firm's primary business (cannibalization)
- Underestimating data infrastructure costs (clean datasets are expensive to maintain)
- Treating one-time data sales as recurring revenue (they're transactional)
- Ignoring data-quality decay (datasets need ongoing maintenance)
- Regulatory missteps (sector regulators move fast)

## Key metrics

- Data revenue as % of total
- Data subscriber count and growth
- Dataset freshness (lag from event to availability)
- Privacy compliance score (audit readiness)
- Customer-level data sharing consent rate
- Data quality scores (completeness, accuracy, consistency)

## Decision questions

- What data do we have that others would pay for, and what's the use case?
- Are we extracting maximum value from data, or just storing it?
- What privacy/regulatory constraints govern our data?
- Could data monetization undermine our primary business?
- Is our data fresh and clean enough to charge premium prices?

## Canonical examples

- **Bloomberg** — financial data terminal; $25K+/year per terminal; massive data moat
- **Experian / Equifax / TransUnion** — credit data providers
- **Nielsen** — TV audience measurement data
- **Foursquare** — location data licensed to brands
- **Telcos** — anonymized mobility data sold to retail/urban-planning
- **Google / Meta** — indirect monetization: user data → ad targeting (page 79)
- **Healthcare data brokers** — anonymized clinical data for pharma research
- **Bloomberg ANYWHERE** — data subscription combining trading + analytics
