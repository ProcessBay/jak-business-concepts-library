# Key Resources

## Definition

Key Resources is the BMC block that identifies the most important assets required to deliver the value proposition — the physical, intellectual, human, and financial resources without which the business cannot operate. Pereira distinguishes four resource types (tangible, intangible, human, financial) and three resource orientations (product-driven, scope-driven, infrastructure-driven), where the orientation determines which resource type dominates. Identifying key resources separates "nice to have" from "without this, we don't have a business."

## Key attributes

- **Four resource types** — Tangible (physical assets, real estate, equipment), Intangible (IP, brand, data, software), Human (talent, leadership, expertise), Financial (cash, credit lines, investor capital)
- **Three business orientations** — Product-driven (R&D, design = key), Scope-driven (broad portfolio = key resources are diverse), Infrastructure-driven (network, platform = key resources are systems)
- **Critical, not exhaustive** — only the resources WITHOUT which the business breaks
- **Linked to activities** — resources enable the key activities
- **Source of competitive advantage** — proprietary resources (brand, IP, exclusive contracts) are moats
- **Cost driver** — most fixed costs trace to acquiring/maintaining key resources

## When to use

- Defining the operating model for a new business
- Diagnosing why scaling is hard (often a key resource constraint)
- M&A due diligence — what resources are we actually buying?
- Strategic re-prioritization — which resources to invest in vs. which to divest
- Talent strategy — if humans are key resources, what's your retention plan

## When NOT to use

- As a complete asset inventory — Key Resources is selective, not exhaustive
- For pure financial analysis (use balance sheet for that)
- In service businesses where the only key resource is people, this block is thin

## Common pitfalls

- Listing every asset (the block should be 3-7 truly critical resources)
- Underestimating intangible resources — brand, data, IP often more valuable than physical
- Ignoring human resources in capability-intensive businesses
- Confusing resources (what you have) with activities (what you do)
- Failing to identify scarcity risk in key resources (single supplier, key person, etc.)

## Key metrics

- Resource utilization rate
- Asset turnover (revenue / total assets)
- ROIC (return on invested capital)
- Intangible asset value (when measurable)
- Talent retention rate in key roles
- Capacity headroom

## Decision questions

- What 3-7 resources, if we lost them, would break the business?
- Which key resources do competitors lack and why is that defensible?
- Are we under-investing in our key resources?
- What single-point-of-failure exists in our resource base?
- Is our business product-, scope-, or infrastructure-driven, and is our resource mix aligned?

## Canonical examples

- **Apple** — intangible: brand, design IP, OS ecosystem; human: design + engineering talent; tangible: retail stores; financial: massive cash reserves (page 181)
- **Amazon** — tangible: fulfillment centers, AWS data centers; intangible: customer data, recommendation algorithms; human: engineers; financial: scale capital (page 185)
- **Netflix** — intangible: content library, viewing data, recommendation engine; financial: capital to fund content; human: data science + product (page 192)
- **Coca-Cola** — intangible: brand, formula; tangible: bottling/distribution infrastructure (page 225)
- **Google** — intangible: search algorithms, data, brand; human: engineering talent; tangible: data centers (page 173)
