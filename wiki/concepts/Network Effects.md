# Network Effects

## Definition

Network effects describe the phenomenon where a product or service becomes more valuable as more people use it. Bharat Anand (Harvard) distinguishes two types: **Direct network effects** — value rises with each additional user of the same type (Facebook becomes more useful as more friends join); **Indirect network effects** — value rises for one segment because another segment grew (Uber riders benefit as more drivers join; Uber drivers benefit as more riders join). Network effects are among the most powerful competitive moats in business: they compound, create winner-take-most dynamics, and make late-mover competitors very hard to dislodge. They also produce switching cost as a side effect.

## Key attributes

- **Two primary types** — Direct (same-side) and Indirect (cross-side)
- **Compound growth** — each user adds value for others, attracting more users
- **Critical mass required** — networks below critical mass have low value; above, value accelerates
- **Winner-take-most dynamic** — networks tend toward concentration (1-2 dominant players per category)
- **Hard to dislodge** — incumbents with strong network effects are very defensible
- **Geographic / category density matters** — Uber needs density per city, not global density
- **Negative network effects exist** — overcrowded marketplaces, congestion, spam dilution

## When to use

- Designing platforms or marketplaces where users interact
- Diagnosing why a network competitor is so hard to beat
- Strategic investment in seeding critical mass (which often means initial losses)
- Recognizing categories ripe for network-effect-driven disruption

## When NOT to use

- Pure single-user products where additional users add nothing (a calculator app)
- Markets where regulation or fragmentation prevents network coalescence
- As a hand-wave excuse for poor product (network effects don't fix bad products)

## Common pitfalls

- Claiming network effects when there are merely scale economies (different thing)
- Underestimating critical mass — losing capital before reaching it
- Failing to seed the right side first (in indirect network effects)
- Letting negative network effects emerge (spam, low-quality content, congestion)
- Confusing virality with network effects (viral = acquisition mechanism; network effect = retention/value mechanism)

## Key metrics

- Correlation of user growth with per-user value (e.g., NPS, retention)
- Retention by network size cohort
- Viral coefficient (often correlated with but distinct from network effects)
- Density per geography / category (where network is "local")
- Cross-side conversion rate (in indirect effects)

## Decision questions

- Are our network effects direct or indirect? (Different implications)
- What's our critical mass per geography / category?
- Where could we seed faster (which side, which segment)?
- Are we letting negative network effects emerge?
- What would a competitor have to do to dislodge our network?

## Canonical examples

- **Facebook, Twitter, Instagram** — direct network effects (more users = more value per user) (page 12)
- **Tinder** — direct network effects (large user base = more matches) (page 12)
- **Uber, Airbnb** — indirect network effects (more drivers/hosts = better for riders/guests, and vice versa)
- **LinkedIn** — both direct (more professionals = more value to professionals) and indirect (more recruiters = paid premium recruiter products)
- **WhatsApp** — direct network effects (people use what their friends use)
- **App stores** — indirect (more developers = more apps = more users = more developers)
- **YouTube** — indirect (more creators = more videos = more viewers = more creators)
