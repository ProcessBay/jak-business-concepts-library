# Function Cost Analysis

## Definition

Function Cost Analysis (FCA) — also called Function Value Analysis (FVA), and in the broader literature, Value Engineering or Value Analysis — was developed by Juryj Sobolev as a structured technique for interrogating every function a product or service performs against the cost of providing it. Pereira's framing: extensive research into the operations and economics of the organization with the sole objective of optimizing expenses, consumer functions, and values. The technique resolves to a binary at each function: **if a function isn't critical or essential, remove it and its associated costs immediately**; **if a function is critical or essential, find the lowest-cost approach to delivering it**. Applied across direct, indirect, intangible, and competitive costs, FCA yields not only unit-cost savings but intangible benefits (safety, simplicity, focus) and durable competitive edge.

## Key attributes

- **Function as the unit of analysis** — not parts, not steps; what the thing *does*
- **Binary decision per function** — eliminate or deliver at lowest cost
- **Four cost domains** — direct, indirect, intangible, competitive
- **Sobolev origin** — predates modern value engineering literature; same intellectual lineage
- **Quality and safety upside** — fewer functions = fewer failure modes
- **Stackable with DFM/DTC** — FCA decides *what* to build; DFM/DTC decide *how* cheaply

## When to use

- Mature products with feature bloat that nobody has audited
- Pre-launch reviews of new products against a cost target
- Service redesign where activities have accreted over time
- Procurement-led cost engineering programs
- Down-market expansion (deciding what to strip)
- After acquisition, to rationalize a combined portfolio's function set

## When NOT to use

- Highly experimental products where function value is still being discovered
- Regulatory-heavy environments where "non-critical" functions may be statutorily required
- Customer-segment products with deeply asymmetric function value (one segment's bloat is another's must-have)
- Early-stage MVPs where function set is intentionally minimal already

## Common pitfalls

- Engineering-led function audits that ignore customer-perceived value
- Eliminating functions that compete with **competitors'** standard offer (creates a perception gap)
- Confusing function with feature — multiple features can serve one function
- Single-pass FCA — functions need to be re-audited as markets evolve
- Killing intangible-value functions because their cost is visible and their value isn't measured
- Using FCA only on COGS and ignoring indirect/overhead functions

## Key metrics

- **Cost per function** — dollar load attributed to each function
- **Function-value ratio** — customer-perceived value / cost
- **Eliminated-function savings** — annualized
- **BOM cost reduction** post-FCA
- **Production and sales-revenue uplift** attributable to the redesign (per Pereira, FCA contributes to both)

## Decision questions

- For each function this product delivers, is it critical or essential — yes or no?
- Where critical, are we delivering it via the lowest-cost approach available?
- Which functions consume direct, indirect, intangible, **and** competitive cost?
- Are intangible benefits being credited fairly, or only cash savings?
- Have we asked customers what they actually value, or are engineers deciding?

## Canonical examples

- **Sobolev's original applications** — Soviet-era manufacturing optimization (foundational case)
- **GE in the 1940s–50s** — Lawrence Miles's parallel value-engineering work, the Western counterpart
- **Automotive cost engineering** — Toyota value analysis programs reducing part count without function loss
- **Consumer electronics** — feature stripping in mid-cycle refreshes to defend margin
- **Hospital service redesign** — clinical-pathway FCA reducing non-value-adding activities
